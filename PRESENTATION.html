Sécurité

Authentification & Contrôle d'accès

L'application utilise deux systèmes d'authentification indépendants avec des tokens JWT stockés dans des cookies httpOnly sécurisés.

🔐

Token Responsable

Cookie : auth_token

  • ✓ Durée de validité : 7 jours
  • ✓ Cookie httpOnly (non accessible JS)
  • ✓ SameSite: lax, Secure en production
  • ✓ Protège les routes /dashboard, /equipements
🛡️

Token Administrateur

Cookie : admin_token

  • ✓ Système entièrement séparé
  • ✓ Cookie httpOnly sécurisé
  • ✓ Protège toutes les routes /admin/*
  • ✓ Accès global multi-sites

Chaque site a une couleur personnalisée stockée en base de données. Elle est appliquée dynamiquement sur la barre de contexte du responsable de site. La variable CSS --site-color est injectée dans le layout pour une cohérence visuelle par site.

📑 Génération PDF côté client

Les rapports sont exportés en PDF via @react-pdf/renderer avec chargement dynamique (no SSR) pour des performances optimales. Le PDF inclut le récapitulatif complet de la situation logistique du site pour la journée.

L'administrateur peut consulter tous les rapports depuis /admin/rapports, avec filtrage par site et date, et export PDF individuel de chaque rapport.

Architecture

Architecture & Organisation du code

Structure du projet Next.js 14 avec App Router, organisée par domaine fonctionnel.

Structure des dossiers
DossierRôle
app/Routage Next.js (App Router) — pages & layouts
app/(responsable)/Groupe de routes responsable (dashboard, sites…)
app/(site)/Site vitrine GI2E public
app/admin/Espace administrateur protégé
components/Composants React réutilisables par domaine
service/Server Actions Next.js (appels API REST sécurisés)
components/pdf/Composants de génération PDF
lib/Utilitaires partagés (utils.ts, cn())
middleware.tsProtection des routes (JWT cookies)
Flux de données
  • 1 Server Actions — Chaque module a ses actions serveur dans service/ (ex: interventions.action.ts). Appels API REST avec Bearer token.
  • 2 TanStack Query — Cache et synchronisation côté client. useQuery pour lecture, invalidation au succès des mutations.
  • 3 État local (useState) — Gestion optimiste des listes (mise à jour locale immédiate avant confirmation serveur).
  • 4 Polling — Données critiques (notifications, situations admin) rafraîchies automatiquement par intervalles.

🏗️ Providers globaux

QueryClientProviderWrapper — Fournit le cache React Query à toute l'application.
ToastProvider — Notifications toast globales (react-hot-toast).
Header1 — Barre de navigation globale avec profil utilisateur.
